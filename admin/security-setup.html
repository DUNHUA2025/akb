資料庫安全設置

啟用 Row-Level Security (RLS) 保護 AKB Salon 資料庫

← 返回後台
⚠️
安全漏洞：RLS 未啟用
Supabase 資料庫的 bookings、 designers、 services、 accounts 表格未啟用 Row-Level Security，任何人知道專案 URL 即可讀寫所有資料。

📋 修復步驟

1
前往 Supabase 儀表板取得 service_role key
開啟下方連結，在 Project Settings → API 頁面找到 service_role JWT token，點擊「Reveal」後複製。
🔗 開啟 Supabase API 設定頁面
2
貼入 service_role key
將複製的 service_role key 貼入下方輸入框（格式：eyJhbGciOi...）。 此 key 僅用於本次 RLS 設置，不會永久儲存。
💡 service_role key 與 anon key 格式相同（都是 JWT），區別在於 payload 的 role 欄位
3
執行 RLS 啟用
點擊下方按鈕，系統將自動在所有資料表上啟用 RLS 並設置適當的存取政策。

執行結果

🔧 最後一步：更新 Render 環境變數

RLS 已啟用。現在後端需要改用 service_role key 才能繼續正常存取資料庫。 請前往 Render 設定環境變數：

1
前往 dashboard.render.com → 選擇 akb-salon-server → Environment
2
新增環境變數：SUPABASE_SERVICE_KEY = 你的 service_role key
3
儲存後 Render 會自動重啟伺服器，後端即可用 service_role key 繞過 RLS

📊 RLS 策略說明

資料表 匿名讀取 匿名寫入 說明
bookings ❌ 封鎖 ✅ 僅插入 客戶可預約，不可看他人記錄
designers ✅ 可讀（在職） ❌ 封鎖 客戶可查看設計師資訊
services ✅ 可讀 ❌ 封鎖 客戶可查看服務項目
accounts ❌ 完全封鎖 ❌ 完全封鎖 帳號資料僅後端可存取
* service_role key（後端）可完整存取所有資料表，繞過 RLS 限制
